Foggy MCP Gateway

Let AI query your Odoo data — securely, with full permission enforcement

What is Foggy MCP Gateway?

Foggy MCP Gateway is an Odoo addon that exposes your business data to AI clients (Claude Desktop, Cursor, VS Code Copilot, etc.) through the Model Context Protocol (MCP).

AI assistants can query your Sales Orders, Purchase Orders, Invoices, Stock Pickings, HR Employees, Partners, and Companies — all while respecting Odoo's native permission system.

Used as a standalone MCP gateway, this addon does not require `openai` or `anthropic`. Those SDKs are only optional dependencies for the built-in AI Chat feature.

Database support for this release is currently scoped to PostgreSQL. MySQL is not yet an officially supported target for this Odoo bridge release and is better treated as a future v1.1 milestone after dedicated validation.

Key Features

🔒

Odoo-Native Security

ir.model.access controls which data models each user can query. ir.rule domains are automatically converted to SQL WHERE clauses and injected server-side — AI clients cannot bypass them.

🏢

Multi-Company

Odoo's multi-company rules (child_of / parent_of) are natively supported via closure table hierarchy queries. Users only see data from their allowed companies and subsidiaries.

📊

7 Business Models

Pre-built query models for Sale Orders, Sale Order Lines, Purchase Orders, Invoices, Stock Pickings, HR Employees, and Partners — with dimensions, measures, and aggregations.

Installation

  1. Install this addon in your Odoo 17 instance
  2. Deploy the Foggy MCP Server (Java, self-hosted) on your network
  3. Configure the Foggy MCP Server URL in Settings → Foggy MCP
  4. Create an API Key for each user in Foggy MCP → API Keys
  5. Connect your AI client to https://your-odoo.com/foggy-mcp/rpc

Data Privacy

No third-party cloud services. This addon communicates only with a Foggy MCP Server that you deploy and control. All data stays within your infrastructure.

  • Odoo ↔ Foggy MCP Server: HTTP on your internal network
  • Foggy MCP Server ↔ PostgreSQL: Direct JDBC connection (read-only queries)
  • HTTPS is strongly recommended for production environments
  • No telemetry, no analytics, no external API calls

Requirements

  • Odoo 17 Community or Enterprise
  • Foggy Engine — embedded Python engine, or external Foggy Java / Python service
  • External Foggy Java server (optional, Java 17+) — Download
  • PostgreSQL 12+ (shared with Odoo, current validated database target)
  • MySQL is not yet an officially supported database target for this release
  • No `openai` / `anthropic` dependency is required for standalone MCP usage
  • Built-in AI Chat optionally requires `openai` and/or `anthropic`
